# Macro Stress Dashboard User Guide

## What This App Is

This dashboard is a monitoring and interpretation system for one specific macro chain:

1. shipping / marine insurance disruption
2. physical oil stress
3. local-currency energy shock for major importers
4. offshore dollar funding stress
5. weaker marginal demand for U.S. Treasuries
6. Treasury plumbing stress
7. Fed intervention pressure
8. inflation persistence and repression risk

It is not a prediction engine. It is a bounded scoring system plus a configured latent-state monitor designed to show where stress is, how it is propagating, and whether it is broadening.

## Current Model Status

The live model was remediated to remove several misleading features from earlier builds.

The current live behavior is:

- regime cards are bounded to `0-100`
- propagation is bounded and capped
- source quality is discounted explicitly
- percentile calculations are tie-aware
- event-driven auction signals are separated from continuously traded market signals
- the latent-state layer is a configured monitor with **relative confidence**, not a calibrated probability model

This is an interpretive system, not a statistically estimated forecasting model.

## How To Read The App

Start in this order:

1. `Regime cards`
2. `Ordering Discipline`
3. `Stagflation Layer`
4. `Physical vs Financial Migration`
5. `Critical Indicators`
6. `Fast-Moving Stress Panel`
7. `Causal Chain`
8. detailed market and domestic panels
9. `Advanced Analytics`

That order matters. The app is designed to keep early physical signals distinct from later funding, Treasury, and Fed-plumbing signals.

## The Three Regime Cards

The cards are absolute bounded scores on a `0-100` scale.

- `0-44`: background
- `45-59`: building
- `60-74`: active
- `75-100`: dominant

The three cards are independent. They do not sum to `100`.

### Sticky Inflation

This card is high when the shock is still mainly an inflation / supply problem:

- oil curve tightness
- freight
- marine insurance
- local-currency importer stress
- inflation expectations
- wage persistence

### Convex Inflation / Funding Stress

This card is high when the shock is migrating into:

- dollar funding stress
- cross-currency basis stress
- SOFR / repo pressure
- rates volatility
- Treasury auction deterioration
- basis-trade stress

### Break / Repression

This card is high when the system is moving into:

- Treasury dysfunction
- Fed backstop pressure
- tax-base erosion
- labor deterioration
- fiscal strain
- repression risk

## Core Scoring Math

### 1. Threshold Normalization

Each indicator is first transformed into a stress score on `0-100`.

For a high-is-bad indicator with warning threshold `W` and critical threshold `C`:

`stress(x) = clamp(((x - W) / (C - W)) * 100, 0, 100)`

For a low-is-bad indicator with warning threshold `W` and critical threshold `C`:

`stress(x) = clamp(((W - x) / (W - C)) * 100, 0, 100)`

Interpretation:

- below warning -> `0`
- at warning -> `0`
- at critical -> `100`
- between warning and critical -> linear interpolation
- beyond critical -> capped at `100`

This is deliberate. Sub-threshold values no longer contribute artificial positive stress the way earlier builds did.

### 2. Source-Confidence Discount

Normalized stress is discounted by source quality before it enters executive composites and propagation features.

Current source-confidence multipliers:

- `live = 1.00`
- `proxy = 0.85`
- `auto = 0.75`
- `manual = 0.65`
- `demo = 0.00`

Composite contribution uses:

`effective_stress = normalized_stress * source_confidence`

This prevents proxy and manual overlays from carrying the same authority as direct live feeds.

### 3. Percentile Rank

Where percentiles are used, the app now uses midpoint ranking for ties:

`percentile(x) = ((count(values < x) + 0.5 * count(values = x)) / N) * 100`

This avoids the earlier first-index bias on flat or repeated values.

## Regime Score Equation

For each regime `r`, direct scoring is a weighted average of normalized indicator stress:

`Direct_r = (sum_i w_{r,i} * s_i) / (sum_i w_{r,i})`

where:

- `w_{r,i}` = configured regime weight for indicator `i`
- `s_i` = normalized indicator stress on `0-100`

Direct regime scores are therefore naturally bounded to `0-100`.

The final regime card score is:

`RegimeScore_r = clamp(Direct_r + PropagationBonus_r, 0, 100)`

## Propagation Layer

Propagation models how stress in one node pressures downstream nodes.

### Causal Nodes

The current causal nodes are:

- `geopolitical_trigger_stress`
- `marine_insurance_stress`
- `oil_physical_stress`
- `expectations_credibility_stress`
- `external_importer_stress`
- `dollar_funding_stress`
- `ust_demand_stress`
- `repo_basis_stress`
- `fed_intervention_stress`
- `household_tax_stress`
- `inflation_repression_stress`

### Current Propagation Parameters

- `iterations = 4`
- `memory = 0.35`
- `activation_floor = 55.0`
- `feedback_gain = 0.22`
- `synergy_gain = 0.0`
- `max_node_boost = 12.0`
- `regime_bonus_cap = 10.0`

### Node Update Logic

Each node has:

- `base_score`
- `amplification`
- `propagated_score = clamp(base_score + amplification, 0, 100)`

At each iteration, an upstream node contributes only if it is above the activation floor.

For an upstream node `u`:

`source_pressure_u = max(0, (propagated_u - activation_floor) / (100 - activation_floor))`

Incoming pressure to node `n` is a weighted average:

`incoming_n = (sum_u edge_{u,n} * source_pressure_u) / (sum_u edge_{u,n})`

Node amplification is then updated as:

`amp_n = clamp(memory * amp_n + feedback_gain * incoming_n * (100 - base_n), 0, max_node_boost)`

This matters for two reasons:

1. propagation is no longer a full second additive scoring layer
2. node amplification is capped

### Regime Propagation Bonus

Each regime has node sensitivities `g_{r,n}`. The raw regime propagation contribution is normalized by the total active sensitivity weight:

`RawBonus_r = (sum_n g_{r,n} * amp_n) / (sum_n g_{r,n})`

Then it is capped:

`PropagationBonus_r = min(RawBonus_r, regime_bonus_cap)`

That cap prevents feedback loops from driving regime cards beyond their scale.

## Auction Stress Architecture

`UST Auction Stress` is no longer a single blunt series.

It is a composite of three subcomponents derived from Treasury auction data:

1. `auction_clearing_stress` (`50%`)
2. `auction_foreign_sponsorship_stress` (`30%`)
3. `auction_issuance_mix_stress` (`20%`)

### Auction Clearing Stress

Uses long-end clearing deterioration:

- bid-to-cover weakness
- high-yield / stop-out deterioration
- dealer absorption

### Auction Foreign Sponsorship Stress

Uses:

- indirect bidder participation
- dealer warehousing pressure

### Auction Issuance Mix Stress

Uses the rolling issuance gap between front-end and long-end issuance, but now scores that gap relative to its own history by percentile instead of using a hard structural bias.

### Auction Card UI

Auction cards are event-native, not continuous market sparklines.

They show:

- current state
- chart window
- last step
- stressed events over `3m` and `6m`
- last recorded step changes

A value of `none` means there were no new stressed step changes in that lookback window, not that auctions did not occur.

## Ordering Discipline

The executive ordering block is there to enforce causal discipline:

1. physical shock
2. importer and household squeeze
3. funding migration
4. Treasury market stress
5. intervention pressure

If later-stage financial variables move without earlier-stage confirmation, the app should be read more cautiously.

## Stagflation Layer

The stagflation block is an interpretive composite of:

- `inflation pressure`
- `growth impairment`
- `policy constraint`

It is not a separate forecast. It exists to answer a specific question:

Is the system becoming dangerous because inflation is staying high while real activity weakens and policy flexibility narrows?

That is the condition under which recursive loops become materially more dangerous.

## Physical vs Financial Migration

This section separates:

- `physical stress`
- `domestic transmission`
- `financial stress`

The app uses this to keep users from overreacting to later-stage funding signals when the underlying physical shock is not confirmed, and vice versa.

## Inflation Expectations And Credibility

The app includes an expectations block built from:

- Cleveland/FRED expected inflation term structure
- 5y5y forward inflation expectations
- survey versus market expectation gap
- expectations entrenchment composite

These feed:

- `expectations_credibility_stress`
- `ust_demand_stress`
- `inflation_repression_stress`

This block is meant to show whether an oil shock is staying temporary or starting to embed in the nominal system.

## Imported-Energy Stress

The app now treats local-currency energy stress as a propagation feature, not just a commodity price feature.

Key measures:

- `oil_in_yen_stress`
- `oil_in_eur_stress`
- `oil_in_cny_stress`
- `external_importer_stress`

This matters because importer stress in Japan, Europe, and China can propagate into:

- weaker marginal UST demand
- tighter dollar funding conditions
- more Fed plumbing pressure

Japan is currently treated as the cleanest and most important transmission path of the three.

### JPY/USD Headline Card

The headline JPY/USD card is now presented as `JPY/USD Funding Stress`.

Internally, the model key remains `jpy_usd_basis` for continuity with the existing regime and propagation logic, but the public card is intentionally labeled more precisely. The live card is a funding-stress construct built from:

- live USD/JPY spot
- official NY Fed SOFR inputs
- Japan short-rate support inputs

This is a live constructed funding signal, not a direct exchange-quoted cross-currency basis print.

## VIX And Tax-Receipts Market Stress

The app now treats VIX as a propagation feature rather than a primary regime anchor.

It contributes through:

- `tax_receipts_market_stress`
- broader risk-premium tightening
- capital-gains-sensitive fiscal pressure

This is a propagation feature, not a direct tax-receipts measure.

## Latent-State Layer

The advanced analytics panel now shows a **configured latent-state monitor**, not a fitted econometric model.

### What It Does

It tracks five latent states:

- `oil_shock`
- `funding_stress`
- `treasury_stress`
- `intervention_pressure`
- `repression_risk`

It uses:

- configured transition matrix
- configured measurement loadings
- trust-based observation noise adjustment
- descriptive episode clustering

### What It Does Not Do

It does **not** currently:

- fit live calibration weights
- fit transition matrices on the fly
- fit observation coefficients on the fly
- run trust-gated calibration blending
- run iterative estimation
- produce statistically calibrated probabilities

### Current Interpretation

The displayed regime percentages are **relative confidence shares**, not posterior probabilities.

They are computed by nonnegative share normalization:

`confidence_r = max(0, score_r) / sum_k max(0, score_k) * 100`

This keeps the advanced layer interpretable without overstating statistical rigor.

### Observation Conditioning

The latent-state layer still adjusts observation trust by:

- current descriptive cluster context
- current dominant latent-state regime

This affects measurement noise only. It does not re-fit the model.

## Data Source Classes

The dashboard uses five source classes:

- `live`
- `proxy`
- `auto`
- `manual`
- `demo`

Interpret them as follows:

- `live`: direct public source
- `proxy`: constructed or indirect but still sourced from live public inputs
- `auto`: document/news/overlay auto-scored from public sources
- `manual`: user override
- `demo`: unavailable / seeded fallback

The executive source note counts headline indicators that are not yet `live`.

## Current Caveats

1. Some secondary non-headline panels may still use proxy sources depending on public-source availability, but the headline card set is intended to resolve to live inputs or live constructed support signals.
2. Public sources can time out or revise history.
3. Documentary overlays are useful, but inherently less clean than market data.
4. The latent-state panel is intentionally conservative in its claims. It is descriptive and configured, not statistically calibrated for public probabilistic use.

## Practical Reading Rules

1. Trust direct physical confirmation more than commentary.
2. Treat proxy inputs as supportive, not decisive.
3. Read funding migration only after checking the physical block.
4. Read break risk only after checking whether auction, liquidity, and Fed-plumbing signals are actually active.
5. If the source note indicates proxy or fallback use, reduce confidence accordingly.

## Summary Of The Live Math

The live model is now:

- bounded
- interpretable
- source-discounted
- tie-aware on percentiles
- capped on propagation
- explicit about confidence versus probability

That makes it suitable as a disciplined macro monitoring tool.

It does not make it an oracle.
